# 🚀 Projects

## ♻️ 시각 장애인을 위한 분리수거 도움 시스템 – Re-트리버

- 유형: 팀 프로젝트
- 역할: 서버 파트 기획 / 논문 작성
- 기술 스택: 서버 구조 설계, API 설계, 데이터 흐름 모델링
- 성과: **학술 논문 은상 수상**

### 프로젝트 개요
시각 장애인의 분리수거 과정에서 발생하는 정보 접근 문제를 해결하기 위해  
클라이언트 요청을 중심으로 안내 데이터를 제공하는 **서버 중심 구조**를 설계한 프로젝트입니다.

### 백엔드 핵심 기여
- 서비스 전체 서버 구조 및 역할 분리 설계
- 클라이언트 요청 → 처리 → 응답 흐름 정의
- 데이터 저장 및 전달 구조를 논문 수준으로 문서화

### 문제 & 해결
- 클라이언트 단에서 모든 로직을 처리할 경우 확장성 및 유지보수 한계 발생  
  → 서버 중심 구조로 책임을 분리하여 구조적 안정성 확보

🏆 수상: 학술 논문 **은상**



## 🚗 스마트폰 기반 특별 구역 불법 주차 AI 감지 및 관리 시스템 (LOTIFY)

- 유형: 팀 프로젝트 (캡스톤 디자인)
- 역할: 팀장 / AI 파트 / 논문 작성
- 기술 스택: Python, YOLO 기반 객체 인식, 영상 데이터 처리
- 성과:
  - **한국정보기술학회 대학생 논문경진대회 우수논문상(동상)**
  - **캡스톤 디자인 동상 수상**

### 프로젝트 개요
스마트폰 촬영 영상에서 차량과 주변 환경을 인식해  
장애인 주차구역·소화전 구역 등 특별 구역의 불법 주차를  
자동으로 탐지하는 **AI 기반 감지 시스템**입니다.

### AI 핵심 기여
- 불법 주차 상황 판단을 위한 객체 인식 모델 설계
- 차량·표지·구역 객체를 구분하는 학습 데이터 구성
- 실제 환경을 고려한 탐지 기준 정의
- 프로젝트 전반 기획 및 팀 리딩 수행

### 문제 & 해결
- 다양한 촬영 각도와 조명 환경에서 탐지 정확도 저하  
  → 데이터 보강 및 기준 조건 조정으로 탐지 안정성 향상

🏆 수상:
- 학회 논문 **우수논문상(동상)**
- 캡스톤 디자인 **동상**



## 🌱 감정 · 회고 · 할 일 기반 자기관리 서비스 – Growlog

- 유형: 개인 프로젝트
- 역할: 기획 / 백엔드 개발 전체
- 기술 스택: 서버 설계, 데이터 모델링, 통계 로직 구현

### 프로젝트 개요
감정·회고·할 일 데이터를 하나의 서버 구조에서 통합 관리하여  
사용자의 행동과 감정 흐름을 함께 분석하는 **백엔드 중심 자기관리 서비스**입니다.

### 백엔드 핵심 구현
- 날짜 기준 통합 데이터 모델 설계
- 감정 기록 / 회고 / 할 일 데이터 저장 구조 구현
- 감정 빈도 및 할 일 달성률 계산 로직 구현
- 통계 데이터를 클라이언트에 전달하는 API 구조 설계

### 문제 & 해결
- 감정·행동 데이터가 분리되어 있어 패턴 분석이 어려움  
  → 서버에서 통합 관리하는 구조로 데이터 활용도 향상
